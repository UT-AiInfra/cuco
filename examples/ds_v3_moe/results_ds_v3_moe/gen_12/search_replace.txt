<DESCRIPTION>
The current bottleneck is that Rank 0 takes 141.8ms because:
1. self_compute (67ms) overlaps with dispatch_wait (67ms) - good
2. remote_compute (67ms) runs AFTER dispatch_wait completes - serialized
3. combine_put waits for ALL compute to finish before starting

The key insight is that self_compute produces output that can be sent via combine_put BEFORE remote_compute finishes. By issuing combine_put for the self-chunk immediately after self_compute completes (while remote_compute is still running), we can overlap the combine communication with remote_compute.

This change:
1. Issues combine_put for the self-chunk right after self_compute completes
2. Runs remote_compute in parallel with the combine transfer
3. Issues combine_put for the remote-chunk after remote_compute finishes
4. Uses a separate signal (signal index 2) for the early self-chunk combine

This should reduce Rank 0's time from ~141ms to ~74ms (self_compute + remote_compute overlapped with combine).
</DESCRIPTION>
